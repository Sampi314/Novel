# AI-Assisted Character Creator — Design

## Goal
Add an in-app character creation modal on the Nhân Vật page that combines manual core input with Claude AI-powered profile generation, saving results to characters.csv and per-character JSON files.

## Architecture
Full-screen modal triggered from CharactersPage. Two-phase workflow: user fills core identity fields, then Claude API generates the full profile (personality, voice, history, arc). User reviews/edits before saving. Browser-side API calls with user-provided Anthropic API key stored in localStorage.

## Modal Structure

### Phase 1: Core Input (User fills in)

| Field | Type | Source |
|-------|------|--------|
| Name (Vietnamese) | text input | user |
| Hán tự (漢字) | text input | user |
| Race/Faction | dropdown | from `data.factions` |
| Role | text input | user |
| Qi Affinity | dropdown | tối cao/cao/trung/âm |
| Power level | slider 1-5 | user |
| Era start | number | user |
| Era end | number (optional) | user |
| Location | dropdown | from `data.locations` |
| Brief concept | textarea | user (1-2 sentences) |

### Phase 2: AI Generation (Claude fills, user edits)

Sections generated by Claude API:

1. **Appearance (外貌)** — First impression, distinctive features, body language
2. **Personality (性格)** — Surface/true/dark personality, 3 core values, core conflict, deepest fear, deepest desire, secret, quirks
3. **Voice (聲音)** — Vocabulary/style, example lines (normal/angry/sad/combat), speech to different audiences
4. **History (歷史)** — Childhood event, turning point, past secret
5. **Character Arc (弧線)** — Starting state → false belief → journey → end state, arc type, price of change
6. **Journey (行程)** — Location sequence (selected from existing locations)

## API Integration

- **Key storage:** User enters Anthropic API key → `localStorage.setItem('cng-api-key', key)`
- **Endpoint:** Direct browser fetch to `https://api.anthropic.com/v1/messages`
- **Model:** claude-sonnet-4-6 (fast, cost-effective for generation)
- **System prompt:** Contents of `.claude/commands/character.md` (embedded in code)
- **User message:** Core fields + concept + existing world context (faction descriptions, era info)
- **Response format:** Structured JSON matching the section fields
- **Error handling:** Show error message if API fails, allow retry

## Data Storage

### characters.csv (append row)
Core fields only: `id,name,han,faction,role,qi_affinity,power,era_start,era_end,location_id,journey`

- `id`: Auto-generated `c{NN}` (next available number)
- `journey`: Pipe-separated location IDs

## Visual Design

- Dark modal overlay with `backdrop-filter: blur(12px)`
- Gold border (`var(--gold)`) and xianxia serif fonts
- Section headers with large dim Hán tự watermarks (身份, 外貌, 性格, etc.)
- Core input fields: dark bg, gold focus glow, EB Garamond font
- AI sections: editable textareas with "AI generated" subtle indicator
- Loading animation: breathing gold glow with "Đang tạo nhân vật..." text
- Save button: gold gradient, hover glow

## Constraints

- Single user, no auth needed
- API key stored in localStorage (acceptable for single-user app)
- CSV append requires dev server running (Vite dev server can't write files — will need a small API endpoint or manual copy)
- Character JSON files written via download + manual placement, OR a small Vite plugin/middleware

## Open Question: File Writing

Since this is a frontend-only app, writing to `characters.csv` and creating JSON files requires either:
1. **Download approach**: Generate CSV row + JSON, user downloads and places them manually
2. **Dev middleware**: Add a small Vite middleware/plugin that accepts POST requests to write files
3. **LocalStorage + sync**: Save everything in localStorage, provide a "Sync to files" button that generates downloadable files

Recommendation: Option 2 (dev middleware) for seamless experience during development.
