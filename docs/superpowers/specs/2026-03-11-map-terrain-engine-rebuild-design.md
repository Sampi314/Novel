# Map Terrain Engine Rebuild — Design Spec

**Date:** 2026-03-11
**Scope:** Replace the Map tab's rendering engine with a new terrain pipeline + Leaflet-based viewer
**Timeline range:** T = 0 to 1,000,000 years across 14 user-defined eras

---

## 1. Architecture Overview

### Scope

Replace only `src/map/`. All other tabs, pages, sidebar, components, data files, and styles remain untouched.

### Stack

| Component | Technology | Purpose |
|---|---|---|
| Map framework | Leaflet.js (~40KB) | Google Maps-grade pan/zoom/touch/momentum |
| Tile rendering | Custom `L.GridLayer` | Dynamic terrain tiles from Web Worker |
| Terrain pipeline | Pure JS + `simplex-noise` | Plate tectonics, heightmap, rivers, biomes |
| React wrapper | `MapViewer.jsx` | Integrates Leaflet into existing tab system |
| Off-thread work | Web Worker | Tile generation without UI blocking |

### Coordinate System

Custom Leaflet CRS mapping world space (0-10,000 x 0-10,000) to pixel space. Existing CSV location coordinates remain compatible.

### Tile Configuration

- Tile size: 256x256 pixels
- Zoom levels: 0-8 (level 0 = entire world in ~4 tiles, level 8 = maximum detail)
- Tiles generated by sampling from pre-computed full-grid keyframe heightmaps (not running the full pipeline per tile)
- Cache: Leaflet's built-in tile caching + manual invalidation on era/theme change

### Web Worker Protocol

Main thread sends to worker:
- `{ type: 'init', keyframeConfigs }` — startup, worker pre-computes all keyframe heightmaps
- `{ type: 'tile', zoom, x, y, T, theme }` — request a tile; worker samples cached heightmaps and returns ImageData

Worker responds:
- `{ type: 'init-complete' }` — keyframes ready
- `{ type: 'tile', zoom, x, y, imageData }` — completed tile (transferred via Transferable for zero-copy)

### Data Flow

```
App.jsx (data, theme, language, currentEra)
  -> MapViewer.jsx
       -> Leaflet map instance
            |-- TerrainTileLayer (custom L.GridLayer)
            |     \-- tileWorker.js (Web Worker)
            |           \-- Terrain Pipeline
            |                 |-- tectonics.js
            |                 |-- heightmap.js
            |                 |-- erosion.js
            |                 |-- hydrology.js
            |                 \-- biomes.js
            |-- RiverLayer (L.GeoJSON)
            |-- TerritoryLayer (L.GeoJSON)
            |-- LocationLayer (L.LayerGroup) — pins + popups
            \-- LeyLineLayer (stub for future)

### Location Pins

- Each location from `data.locations` rendered as a Leaflet marker
- Pin icon styled by location type (city, sect, sacred site, etc.)
- Click opens a Leaflet popup with: name (in selected language), type, territory
- Pins filtered by era visibility (only shown if location exists in current T range)
- `mapZoomTarget` prop triggers `map.flyTo(target)` for cross-tab navigation
```

## 2. Terrain Generation Pipeline

### Stage 1 — Plate Tectonics

- Generate 7-12 tectonic plates using Voronoi seed points
- Plate boundaries use constrained random walks + Perlin noise displacement (never straight edges)
- Each plate has a velocity vector determining drift direction over T
- Collision boundaries: raise heightmap -> elongated fractal mountain ridges
- Divergent boundaries: lower heightmap -> rift valleys, trenches, new ocean floor
- Transform boundaries: lateral shear zones
- Over 1M years: supercontinent at T=0, fragmented continents by T=1,000,000
- Plate positions pre-computed at each of 14 era keyframes; intermediate T values interpolated

### Stage 2 — Heightmap Generation

- Simplex Noise + Fractal Brownian Motion (fBm) at 6-8 octaves
- Domain warping (noise-fed-into-noise) for organic, non-repetitive landforms
- Tectonic influence layer added on top: collisions boost height, rifts depress it
- Hydraulic erosion simulation (thermal + rainfall):
  - Water droplets flow downhill carrying sediment
  - Carves valleys, smooths peaks, creates drainage patterns
  - ~50,000 erosion iterations per keyframe on the full-resolution heightmap grid
- Continent mask ensures coherent land masses
- Per-keyframe: erosion amount increases with T (older = more weathered)

**Keyframe pre-computation strategy:**
- All keyframe heightmaps are computed once at app startup (background Web Worker)
- Full pipeline runs on a 1024x1024 grid per keyframe (~14 keyframes)
- Estimated: ~2-5 seconds total for all keyframes on modern hardware
- Results cached in memory (Float32Arrays); optionally persisted to IndexedDB for instant reload
- Individual tiles sample from the cached full-grid heightmap (no per-tile pipeline runs)
- Loading indicator shown during initial keyframe computation

### Stage 3 — Coastlines & Hydrology (D8 Algorithm)

- Sea level threshold on heightmap defines coastlines
- D8 flow direction: every cell flows to lowest of 8 neighbors (mandatory algorithm)
- Flow accumulation: count upstream cells flowing into each cell
- Rivers form where accumulation exceeds threshold
- Strict rules (verified programmatically):
  - Rivers ALWAYS flow downhill (height strictly decreases along path)
  - Rivers ALWAYS terminate at ocean or inland sea (never mid-land)
  - Rivers merge downstream (tributary system)
  - Rivers widen source-to-mouth (stroke width = f(accumulation))
- River valleys carved into heightmap after generation (erosion feedback)
- Meander simulation at later keyframes: bend curvature increases over T, oxbow lakes form at cutoff threshold
- Delta formation: sediment deposition at river mouths builds branching fan deltas over T
- Rendered as smooth cubic Bezier curves with variable width

- Temperature: decreases with latitude + altitude

## 3. Keyframe System

### Morphable Heightmaps

- 14+ pre-computed heightmaps (one per era, plus extras at major geological events)
- For any T value, find the two nearest keyframes and blend via weighted interpolation
- Blending is per-pixel weighted average (cheap computation)
- Tile cache keyed by `(zoom, x, y, T_rounded)` — invalidated when T crosses a keyframe boundary

### What Evolves With T

- Continental positions (plate drift)
- Coastline shapes (erosion, sea level changes)
- Mountain heights (collision uplift, then erosion)
- River courses (meander simulation, oxbow lakes, delta growth)
- Biome boundaries (climate shifts from continent movement)

### Era Structure

- Currently 7 eras defined in `world.json` (existing data). User plans to expand to 14 eras.
- Era names, themes, and T boundaries defined in data files (not hardcoded)
- Engine supports any number of eras — add/remove by editing configuration
- Current `world.json` era years will be remapped to the new T=0 to 1,000,000 range
- Each era in the data file specifies: `{ name_vi, name_zh, startYear, endYear, description }`

## 4. No Straight Lines Rule

### Core Constraint

Every boundary, border, and overlay edge in the entire map system must use organic, fractal displacement. No straight segments longer than 20 pixels anywhere.

### Techniques

1. **Terrain-locked edges** — follow rivers, ridgelines, or coastlines from heightmap
2. **Fractal displacement** — recursive midpoint displacement on segments > N pixels
3. **Noise perturbation** — Simplex noise offset perpendicular to edge
4. **Micro-jitter** — pixel-level randomness

### Enforcement

- Shared `fractalizeEdge(points, noiseScale, iterations)` utility function
- Every layer must call this before rendering any polygon or polyline
- Quality gate: no boundary segment > 20px without displacement (verified visually)

### Applies To

- Biome boundaries
- Faction/race territory borders
- Tectonic plate boundaries
- Coastlines
- Qi heatmap zones
- All future overlay edges

## 5. Integration With Existing App

### Props Interface

```jsx
<MapViewer
  data={data}              // locations, rivers, territories, leyLines
  theme={theme}            // 'dark' | 'light'
  language={language}      // 'vi' | 'zh' | 'vi-zh'
  onLanguageChange={setLanguage}  // callback to update App-level language state
  isVisible={activeTab === 'map'}
  mapZoomTarget={mapZoomTarget}  // {x, y} from other pages (fly-to-location)
/>
```

Note: `currentEra` is internal state within MapViewer (controlled by the era timeline slider). App.jsx does not own era state.

### Mounting Strategy

Map stays mounted via `display:none` when tab is inactive (preserves Leaflet zoom/pan state). Leaflet instance created once on mount, never destroyed.

### Language Toggle

New `language` state in App.jsx (persisted to localStorage):
- `'vi-zh'` — Vietnamese primary + Chinese secondary (default)
- `'vi'` — Vietnamese only
- `'zh'` — Chinese only

Toggle button in map's top-right corner (Leaflet control). Affects all labels: location pins, territory names, river names, era labels.

### Era Timeline Slider

- Continuous slider from T=0 to T=1,000,000 below the map
- 14 era labels as tick marks along the slider
- Dragging smoothly interpolates terrain between keyframes
- Current era name displayed in selected language

### Changes to App.jsx

- Add `language` state: `const [language, setLanguage] = useState(localStorage.getItem('cng-language') || 'vi-zh')`
- Add `language` to localStorage on change
- Pass `language` and `setLanguage` props to MapViewer
- Pass existing `mapZoomTarget` prop to MapViewer (already exists in App.jsx)
- Remove `currentEra` prop if currently passed (era is now internal to MapViewer)
- Import path unchanged: `src/map/MapViewer.jsx`

### Mobile

- Leaflet handles touch natively (pinch-zoom, momentum pan, double-tap)
- Language toggle accessible from map corner
- Era slider touch-draggable
- No custom touch handlers needed

## 6. File Structure

```
src/map/
|-- MapViewer.jsx              # React wrapper for Leaflet
|-- TerrainTileLayer.js        # Custom L.GridLayer
|-- LanguageToggle.jsx         # vi / zh / vi-zh selector
|
|-- pipeline/                  # Terrain generation (Web Worker)
|   |-- tectonics.js           # Plate simulation, random walk boundaries
|   |-- heightmap.js           # Simplex + fBm + domain warping
|   |-- erosion.js             # Hydraulic + thermal erosion
|   |-- hydrology.js           # D8 flow direction, river extraction
|   |-- biomes.js              # Whittaker model -> 34 biomes
|   |-- keyframes.js           # Pre-compute 14 era heightmaps
|   |-- interpolator.js        # Blend between keyframes for any T
|   \-- renderer.js            # Heightmap -> colored tile ImageData
|
|-- layers/                    # Leaflet overlay layers
|   |-- RiverLayer.js          # Dynamic rivers per era (GeoJSON)
|   |-- TerritoryLayer.js      # Faction borders (from world.json pts, fractalized)
|   |-- LocationLayer.js       # Pins + labels
|   \-- LeyLineLayer.js        # Stub for future (animated spiritual veins)
|
|-- utils/
|   |-- noise.js               # Simplex 2D/3D + fBm
|   |-- colors.js              # Biome color palettes (dark/light)
|   |-- crs.js                 # Custom Leaflet CRS (0-10000 world coords)
|   \-- fractalEdge.js         # fractalizeEdge() — no straight lines
|
\-- workers/
    \-- tileWorker.js          # Web Worker for pipeline execution
```

### Deleted

All of current `src/map/` (MapViewer.jsx, TileManager.js, WorldSeed.js, generators/, layers/, utils/, workers/).

Note: existing `RoadGenerator.js` and `RoadLayer.js` (recently added) are intentionally deleted. Road/trade route rendering is out of scope for this rebuild and will be re-implemented in a future iteration with A* pathfinding.

### New Dependencies

- `leaflet` (~40KB) — map framework
- `simplex-noise` (~5KB) — Simplex noise implementation for terrain generation

### Untouched

Everything outside `src/map/`: App.jsx, Sidebar, all pages, components, CSS, data files.

## 7. Quality Gates

Must pass before shipping:

- [ ] No overlay boundary contains a straight segment > 20px
- [ ] No river runs uphill (verified programmatically via heightmap gradient check)
- [ ] No river terminates mid-land (every river reaches coast or inland sea)
- [ ] Mountain ranges are elongated curved ridges (not scattered peaks)
- [ ] Coastlines have visible fractal complexity (bays, peninsulas, fjords at zoom)
- [ ] Rivers widen from source to mouth (never uniform width)
- [ ] Continental positions visibly shift when scrubbing T across eras
- [ ] Biome boundaries are organic/fractal (no hard lines)
- [ ] Terrain has visible geographic variety: distinct mountain ranges, valleys, plains, and coastline indentation (not uniform noise)
- [ ] Leaflet pan/zoom/pinch smooth at 60fps
- [ ] Dark and light themes both render correctly
- [ ] Language toggle switches all labels without page reload
- [ ] Map integrates cleanly with existing App.jsx tab system
- [ ] Tectonic plate boundaries show visible fractal irregularity
- [ ] At later eras, oxbow lakes visible along major rivers
- [ ] At later eras, river deltas visible at coastlines
